# SDK (coming soon)

A developer SDK is on the Velfi roadmap. It will let other applications build on Velfi's payment primitives and agent directly.

{% hint style="warning" %}
The SDK is not yet available. This page describes what's planned. Details will be finalized and documented when it ships.
{% endhint %}

### What the SDK aims to offer

* **Payment primitives** — integrate send, split, stream, schedule, and escrow flows into your own app.
* **Agent access** — let your users instruct payments in plain language, powered by Velfi's agent.
* **`.vel` identity** — resolve and use `.vel` names within your product.
* **Non-custodial by default** — the same security model: your users approve their own payments and hold their own keys.

### Who it's for

Developers building payment experiences, fintech products, or apps that need programmable money — who want the simplicity of Velfi's agent and the safety of self-custodial, on-chain settlement, without building it all from scratch.
